uBlock Orgin 浏览器去广告

发表于 2025-08-06 更新于 2025-08-07 分类于 Chrome ， 浏览器插件
本文字数： 1.5k 阅读时长 ≈ 2 分钟

简介来源于：Github/gorhill

uBlock Origin (uBO) 是一款适用于 Chromium 和 Firefox 的高效 CPU 和内存广谱内容拦截器。它默认使用 EasyList、EasyPrivacy、Peter Lowe 的黑名单、在线恶意 URL 黑名单和 uBO 过滤列表来拦截广告、跟踪器、挖矿程序、弹出窗口、烦人的反拦截器、恶意软件网站等。此外，还有许多其他列表可供选择，可以拦截更多内容。uBO 还支持 Hosts 文件。uBO 使用 EasyList 过滤语法，并扩展了该语法以支持自定义规则和过滤器。如果您认为 uBO 拦截过多，可以轻松取消选择任何预选的过滤列表。作为参考，Adblock Plus 安装时默认仅启用 EasyList、ABP 过滤器和可接受广告。需要注意的是，使用拦截器并非盗窃。不要被这种令人毛骨悚然的想法所蒙蔽。“拦截=盗窃”的最终逻辑后果是将不可剥夺的隐私权定为犯罪。广告，无论是否“非侵入式”，都只是您访问大多数网站时进入浏览器的侵犯隐私手段的可见部分。uBO 的主要目标是帮助用户消除这些侵犯隐私的方法，以欢迎那些不希望使用更多技术手段的用户。

问题描述：

问题描述：Chromium内核的Chrome和Edge浏览器在内核更新到较新版本后无法获取uBlock Orgin插件的支持。

Chrome 自 2024 年 6 月（版本 127）起逐步过渡到 Manifest V3，并计划在 2025 年 6 月完全移除对 Manifest V2 的支持。自 2024 年 10 月起，Chrome 开始逐步禁用 Manifest V2 扩展，包括 uBlock Origin，导致许多用户收到“此扩展不再受支持”的警告。

uBlock Origin 已被 Chrome 网上应用店标记为“已弃用”，无法通过常规方式直接安装。开发者 Raymond Hill 明确表示不会推出 Manifest V3 版的完整 uBO，因为 V3 的限制（如缺少动态过滤、自定义规则等）无法满足 uBO 的核心功能需求。

uBlock Origin 已被 Chrome 网上应用店标记为“已弃用”，无法通过常规方式直接安装。

在你尝开启浏览器插件的开发者模式使用本地文件压缩包文件进行安装时会提醒：

未能成功加载扩展程序
错误
无法安装扩展程序，因为它使用了不受支持的清单版本。无法加载清单。

具体信息参考来源：Reddit

临时解决方案

实验性功能开启

以Chrome浏览器为例，打开浏览器，输入chrome的flag开启链接，在浏览器栏内输入以下网址进入到Chrome的实验功能界面并找到有关manifest相关的配置项，从【Default】修改到【Enabled】

重启浏览器

随后会提示重启生效，点击重新启动

资源获取

在此链接内找到最新的release文件，下载有Chrome标识的文件，截止到2025年的8月6日，最新的文件版本为Pre-release 1.65.1b2，后续随着浏览器版本的更新可能此方法就不再适用，请根据现有信息自行评估此方法的可用性。

下载文件后解压，解压后记好文件的解压位置，后续要用到

导入文件

按照图示，输入插件的管理地址，或者直接在浏览器右侧的设置里进入

开始选择文件，定位到可以看到程序文件的目录，例如我这里的文件夹uBlock0.chromium

忽略manifest低版本提示

此时扩展将会被安装且会提示错误，点击【移除】→【全部清除】

此时即可正常使用并过滤广告。
